\begin{slide}
\slidetitle{Algebraic modeling language}
\small
\zimpl is an algebraic modeling language 
for mixed integer programming problems.

Modeling languages allow to formulate a
mathematical programming problem in 
a way similar to the original mathematical formulation and 
automatically generate input for a corresponding solver.

\begin{itemize}
\item Problems are represented in a declarative way
\item Separation between problem definition and solution process
\item Separation between problem structure and data.
\end{itemize}

Algebraic modeling languages typically operate on
contraints consisting of variables indexed by sets connected to
parameters by algebraic expressions.

\begin{slide}
%\slidetitle{Modeling language}
\lstset{language=zimpl,%
basicstyle=\sffamily\scriptsize,%
numberstyle=\sffamily\tiny\color{siennabrown},stepnumber=0}
\begin{lstlisting}{}
# This model solves Sudoku puzzels.
param p := 3;

set L := { 1 .. p*p };
set M := { 1 .. p};

var x[L*L*L] binary;

# file: row col value
set F := { read "sudoku.dat" as "<1n,2n,3n>" comment "#" };

subto dots: forall <i,j> in L*L do sum <k> in L : x[i,j,k] == 1;
subto rows: forall <j,k> in L*L do sum <i> in L : x[i,j,k] == 1;
subto cols: forall <i,k> in L*L do sum <j> in L : x[i,j,k] == 1;
subto sqrs: forall <m,n,k> in M*M*L do 
   sum <i,j> in M*M : x[(m-1)*p+i,(n-1)*p+j,k] == 1;

# Fix the fixed values
subto fixed: forall <i,j,k> in F do x[i,j,k] == 1;

\end{lstlisting}

\slidetitle{Modeling language}
\small
For $a,b,c\in\ZZ$, 
$a\in [-15,15], b\in [-10,20], c\in [-20,10]$,

maximize $5 a + 3 b + c$
subject to:
\begin{equation*}
\begin{array}{rl}
\text{If}&(a \neq b \text{ and }(|a - b| = 3 c \text{ or } c - a \geq 0))\\
 \text{then}&a + b + c \geq 7\\
 \text{else}& a + b\leq 1
\end{array}
\end{equation*}

\bigskip
{\color{blue}can be modeled as an IP, but this very laborious\ldots}

\begin{lstlisting}[frame=t]{}
var a integer >= -15 <= 15;
var b integer >= -10 <= 20;
var c integer >= -20 <= 10;
maximize obj: 5 * a + 3 * b + c;
subto c1:

      then a + b + c >= 7 
      else a + b <= 1 end;
\end{lstlisting}

\slidetitle{{\large\zimpl}}
%{\large\zimpl} is
\begin{itemize}
\setlength{\itemsep}{0pt}
\item is freely available with C source code under the GPL.
\item is highly portable (*NIX, Windows, MacOS-X).
\item is solver independend.
\item is available as a library.
\item can be used standalone or linked to a solver.
\item is pretty stable. 
\item has been used in several lectures and industry projects.
\item can count correctly.
\end{itemize}

\begin{slide}
\slidetitle{Freely available}

In contrast to most other modeling languages, {\color{blue}\zimpl is 
freely available in source code under GPL.}

This is very convienent for teaching, as all students can use Zimpl
on their laptops and at home. 

\zimpl is highly portable and {\color{blue}compiles under Linux, Windows, MacOS-X,
Solaris, Tru64, IRIX, HP-UX, \ldots}

Only dependend on the GMP (GNU Multi Precision Library).

Download at {\color{red}http://zimpl.zib.de}, binaries and a user
guide are available.

\end{slide}

\slidetitle{Solver independend}

\zimpl used standalone generates either MPS or LP files, which can be
processed by most solvers.

\zimpl can write SOS and ORD files for CPLEX.

It is strait forward and easy to add further formats\\
(as far as \zimpl is concerned ;-)

\begin{minipage}{0.5\textwidth}
$$
\begin{array}{lcl}
v\neq w   & \Leftrightarrow & b^+ + b^- = 1\\
v =   w   & \Leftrightarrow & b^+ + b^- = 0\\
v \leq  w & \Leftrightarrow & b^+ = 0\\
v <     w & \Leftrightarrow & b^- = 1\\
v \geq  w & \Leftrightarrow & b^- = 0\\
v >     w & \Leftrightarrow & b^+ = 1 
\end{array}
$$
\end{minipage}

wobei 
$x=v-w$ für $v,w\in\ZZ$ mit $l_x=l_v-u_w$ und $u_x=u_v-l_w$ gilt, falls
zwei Operanden beteiligt sind.

\begin{slide}
\sffamily
{\color{blue}Modellierung von vif mittels big-M}

\medskip
\hrule

\bigskip
Gegeben eine Variable $r\in\{0,1\}$ und eine Bedingung 
$$\sum a_i x_i\leq b$$ 
mit $-\infty\leq l_i \leq x_i \leq u_i\leq\infty$ und
$$U=\sum_{a_i<0} a_i l_{i} + \sum_{a_i>0} a_i u_{i}<\infty$$

Die Bedingung braucht nur dann erfüllt sein, wenn $r=1$:

\begin{equation*}
\sum a_i x_i + (U-b)r\le U\label{eqn:cond-exec}
\end{equation*}

\end{slide}
